What is curl?

curl (Client URL) is a powerful command-line tool used to transfer data to or from a server using various protocols including HTTP, HTTPS, FTP, and many others. It's one of the most essential tools for developers, system administrators, and anyone working with APIs or web services.

Whether you're testing REST APIs, downloading files, debugging web applications, or automating data transfers, curl is your go-to tool for making network requests from the terminal.

1. Installation

curl is pre-installed on most Linux distributions and macOS. To check if it's installed:

curl --version

Installing on Different Systems:

# Ubuntu/Debian sudo apt update sudo apt install curl # CentOS/RHEL/Fedora sudo yum install curl # or sudo dnf install curl # macOS (if not already installed) brew install curl # Arch Linux sudo pacman -S curl

2. Basic Syntax

The basic syntax of curl is straightforward:

curl [options] [URL]

Simplest Example:

Example: Fetch a Web Page
curl https://example.com

This command fetches the HTML content of example.com and displays it in your terminal.

3. Common Options and Flags

curl has numerous options. Here are the most commonly used ones:

Option Description
-o, --output Write output to a file instead of stdout
-O, --remote-name Save with the remote file name
-L, --location Follow redirects
-i, --include Include response headers in output
-I, --head Fetch headers only (HEAD request)
-X, --request Specify HTTP method (GET, POST, PUT, DELETE, etc.)
-H, --header Add custom header to request
-d, --data Send data in POST request
-u, --user Server user and password (user:password)
-v, --verbose Make the operation more verbose
-s, --silent Silent mode (no progress bar)
-k, --insecure Allow insecure SSL connections

4. HTTP Methods

curl supports all standard HTTP methods. Here's how to use them:

GET Request (Default)

curl https://api.example.com/users

POST Request

curl -X POST https://api.example.com/users \ -H "Content-Type: application/json" \ -d '{"name": "John Doe", "email": "john@example.com"}'

PUT Request

curl -X PUT https://api.example.com/users/123 \ -H "Content-Type: application/json" \ -d '{"name": "Jane Doe"}'

DELETE Request

curl -X DELETE https://api.example.com/users/123

PATCH Request

curl -X PATCH https://api.example.com/users/123 \ -H "Content-Type: application/json" \ -d '{"status": "active"}'

5. Practical Examples

Download a File

Save with custom name:
curl -o myfile.zip https://example.com/file.zip
Save with original filename:
curl -O https://example.com/file.zip

Follow Redirects

Many URLs redirect. Use -L to follow them:
curl -L https://github.com

Send Headers

Add authentication token:
curl -H "Authorization: Bearer YOUR_TOKEN" \ https://api.example.com/protected

POST JSON Data

Send JSON to an API:
curl -X POST https://api.example.com/data \ -H "Content-Type: application/json" \ -d '{ "username": "testuser", "password": "testpass", "email": "test@example.com" }'

Upload a File

Upload using multipart/form-data:
curl -X POST https://api.example.com/upload \ -F "file=@/path/to/file.pdf" \ -F "description=My Document"

Basic Authentication

Authenticate with username and password:
curl -u username:password https://api.example.com/protected

View Response Headers

See full response including headers:
curl -i https://api.example.com/users
Headers only (no body):
curl -I https://api.example.com/users

Test API Performance

Measure response time:
curl -w "\nTime Total: %{time_total}s\n" \ -o /dev/null -s https://api.example.com

Download Multiple Files

Download files with pattern:
# Download file1.txt through file5.txt curl -O https://example.com/file[1-5].txt # Download files with different names curl -O https://example.com/{file1,file2,file3}.txt

6. Advanced Usage

Save Cookies

curl -c cookies.txt https://example.com/login

Use Saved Cookies

curl -b cookies.txt https://example.com/dashboard

Set User Agent

curl -A "Mozilla/5.0 (Windows NT 10.0; Win64; x64)" \ https://example.com

Limit Download Speed

curl --limit-rate 100K -O https://example.com/largefile.zip

Resume Interrupted Download

curl -C - -O https://example.com/largefile.zip

Use Proxy

curl -x http://proxy.example.com:8080 https://api.example.com

Verbose Mode for Debugging

curl -v https://api.example.com/users

This shows detailed request/response information including headers, SSL handshake, etc.

Save Response to File with Progress

curl -# -o output.html https://example.com

Timeout Settings

# Connection timeout (10 seconds) curl --connect-timeout 10 https://api.example.com # Maximum time for operation (30 seconds) curl --max-time 30 https://api.example.com

7. Best Practices

üí° Security Tip

Avoid using -k (--insecure) in production environments as it disables SSL certificate verification, making your connection vulnerable to man-in-the-middle attacks.

1. Use HTTPS When Possible

Always prefer HTTPS over HTTP for secure data transmission:

# Good curl https://api.example.com # Avoid (unless necessary) curl http://api.example.com

2. Handle Errors Properly

Use --fail to ensure curl returns non-zero exit code on HTTP errors:

curl --fail https://api.example.com/resource || echo "Request failed"

3. Store Credentials Securely

Don't hardcode passwords in scripts. Use environment variables:

# Bad curl -u user:password123 https://api.example.com # Good curl -u $API_USER:$API_PASSWORD https://api.example.com

4. Use Configuration Files

For complex requests, create a config file:

# Create ~/.curlrc user-agent = "MyApp/1.0" header = "Accept: application/json" # Use it curl https://api.example.com

5. Combine with Other Tools

curl works great with jq for JSON processing:

# Pretty print JSON response curl -s https://api.example.com/users | jq '.' # Extract specific field curl -s https://api.example.com/users | jq '.[] | .name'

6. Test Before Scripting

Always test curl commands manually before adding them to scripts:

# Test with verbose mode first curl -v https://api.example.com/endpoint # Once confirmed working, use in script #!/bin/bash response=$(curl -s https://api.example.com/endpoint) echo "$response"

Summary

The curl command is an indispensable tool for anyone working with web technologies, APIs, or network services. Its versatility allows you to perform simple file downloads to complex API interactions with custom headers, authentication, and data payloads.

Key takeaways:

  • curl is a command-line tool for transferring data using various protocols
  • It supports all HTTP methods (GET, POST, PUT, DELETE, PATCH, etc.)
  • You can customize requests with headers, authentication, and data
  • It's perfect for testing APIs, downloading files, and automating web tasks
  • Always follow security best practices when handling credentials

Start with simple commands and gradually explore advanced features as you become more comfortable. The more you use curl, the more you'll appreciate its power and flexibility!
